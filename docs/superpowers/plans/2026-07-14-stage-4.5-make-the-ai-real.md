# Stage 4.5 — Make the AI Real

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Mandate's AI half actually execute — a real Granite run through watsonx, real Docling-parsed policy documents, Granite explanations on screen — and close the trust-model gaps (reputation that persists, an agent that learns from being blocked) that a judge will otherwise find.

**Architecture:** One shared Granite client replaces the two duplicated watsonx call sites. The agent gains an optional *policy briefing* (a suggestion) and a *blocked-proposal memory* (in-mission learning), while the evaluator remains the only thing that can actually stop it. Reputation moves out of per-mission memory into the `trust_ledger` table, keyed by agent configuration version, so it survives missions and resets when the prompt or model changes. Every decision the evaluator makes is exported as labelled preference data — the audit trail doubles as a training set.

**Tech Stack:** Next.js 15, TypeScript, Vitest, Drizzle + PostgreSQL 16 (Docker, host port **5433**), `@ibm-cloud/watsonx-ai`, IBM Granite 3.3, Docling (Python 3.11).

## Global Constraints

- **The evaluator stays pure.** No I/O, no LLM calls, no clock, no randomness in `src/engine/evaluate.ts`. Same inputs → same Decision. (`AGENTS.md`)
- **Granite proposes and explains. It never decides.** A Granite explanation must be stored *alongside* the evaluator's deterministic explanation, never overwrite it.
- **`trust_ledger` is append-only.** No UPDATE or DELETE path may be introduced for it.
- **Evaluator and golden-path tests run with no database and no network.** Replay-mode proposals only.
- **The golden path is the CI gate.** `src/engine/golden-path.test.ts` must stay green through every task. If a change cannot survive it, it does not ship. (`SPEC.md`)
- **Definition of done, every task:** `npx tsc --noEmit`, `npm run lint`, `npm test`, `npm run build` all pass locally. CI must be green before a task is considered complete.
- **Next.js 15 gotcha:** dynamic API route `params` is a Promise — `{ params }: { params: Promise<{ id: string }> }` with `const { id } = await params`.
- **Postgres is on host port 5433**, not 5432. A native Windows PostgreSQL squats on 5432.
- **Log every session in `BOB_USAGE.md`** (date, goal, produced, kept/changed).

## Feasibility assessment — read this before starting

Three things could sink this stage. All three have mitigations, and one changes the schedule.

### RISK 1 — Live Granite may not produce the 7-step narrative (HIGH)

Live mode has **never run once**. The orchestrator marches through a fixed 7-step `agentSequence` and takes whatever `propose()` returns. There is no guarantee Granite returns `compare_vendors` at step 3, or that it proposes CheapTech at step 6 — and step 6 **is the demo**.

**Mitigations, in order:**
1. The Sourcing Agent prompt already carries deliberate cost pressure (`agents.ts:21`). Strengthen it if needed.
2. Task 5 adds the **expected phase** to the prompt, so Granite is told what kind of action this step wants.
3. Task 4 validates and retries malformed proposals.
4. SPEC already says record **the best of several live runs**. Budget for 5–10 runs.
5. **The demo video uses replay mode regardless** (SPEC mandates this). Live mode needs to work *once*, well, to record fixtures from. It does not need to be reliable.

**Budget 2 days of prompt iteration.** This is the single largest schedule risk.

### RISK 2 — Docling first run downloads ~1 GB of layout models (MEDIUM)

`pip install docling` then the first `DocumentConverter().convert()` pulls layout/table models. Python 3.11.0 is present. Offline, one-time, not in the request path. Fine, but do it early — not the night before.

### RISK 3 — Risk-weighted reputation would break the golden path (RESOLVED — DO NOT BUILD)

The audit suggested weighting reputation by risk class so it can't be farmed on zero-risk reads. **On inspection this is mostly already handled, and building it would break the CI gate:**

- Golden-path step 3 promotes PROBATION → SUPERVISED off three *read-only* actions. If reads earned nothing, no promotion, and the golden path fails.
- But PROBATION → SUPERVISED buys very little: **every commercial action still reaches a human in both bands** (APPROVAL vs REVIEW). No money moves unsupervised.
- The dangerous band, TRUSTED, **already requires 2 approved spend events** — that *is* risk-weighted evidence, for the band that matters.

**Decision: do not implement risk weighting.** Document the reasoning in the README instead (Task 12). It is a talking point, not a defect.

```typescript
/**
 * The only place a watsonx client is constructed.
 *
 * Previously agents.ts and explain.ts each built their own, each hardcoding
 * `ibm/granite-13b-chat-v2` — a model IBM has withdrawn. Centralising it means
 * the model ID is configured in exactly one place and can be swapped by env var
 * when IBM retires the next one.
 */

const DEFAULT_MODEL_ID = 'ibm/granite-3-3-8b-instruct';
const DEFAULT_URL = 'https://us-south.ml.cloud.ibm.com';

export class GraniteNotConfiguredError extends Error {
  constructor() {
    super(
      'Granite is not configured. Set WATSONX_API_KEY and WATSONX_PROJECT_ID in .env ' +
        'to run in live mode. Replay mode needs neither.'
    );
    this.name = 'GraniteNotConfiguredError';
  }
}

export function getModelId(): string {
  return process.env.WATSONX_MODEL_ID || DEFAULT_MODEL_ID;
}

export function isGraniteConfigured(): boolean {
  return Boolean(process.env.WATSONX_API_KEY && process.env.WATSONX_PROJECT_ID);
}

/** Single-turn chat with Granite. Returns the raw text. */
export async function graniteChat(
  prompt: string,
  opts: { maxTokens?: number; temperature?: number } = {}
): Promise<string> {
  if (!isGraniteConfigured()) throw new GraniteNotConfiguredError();

  const { WatsonXAI } = await import('@ibm-cloud/watsonx-ai');
  const { IamAuthenticator } = await import('ibm-cloud-sdk-core');

  const client = WatsonXAI.newInstance({
    version: '2024-05-31',
    serviceUrl: process.env.WATSONX_URL || DEFAULT_URL,
    authenticator: new IamAuthenticator({ apikey: process.env.WATSONX_API_KEY! }),
  });

  const response = await client.textChat({
    modelId: getModelId(),
    projectId: process.env.WATSONX_PROJECT_ID!,
    messages: [{ role: 'user', content: prompt }],
    maxTokens: opts.maxTokens ?? 500,
    temperature: opts.temperature ?? 0.7,
  });

  const text = response.result.choices[0]?.message?.content?.trim();
  if (!text) throw new Error('Granite returned an empty response');

  return text;
}
```

Create `src/trust/version.ts`:

```typescript
import { createHash } from 'crypto';

/**
 * Identifies an agent CONFIGURATION: role + model + prompt.
 *
 * Reputation binds to this, not to a mission and not to an "instance". A
 * stateless LLM has no instances — two copies of the same prompt on the same
 * model are the same function, and share one track record.
 *
 * Change the prompt or the model and this changes, which resets reputation to
 * zero. That is correct: you tested v1, you trust v1. v2 is a different
 * program and your evidence does not carry over.
 */
export function agentVersion(role: string, modelId: string, prompt: string): string {
  return createHash('sha256')
    .update(`${role} ${modelId} ${prompt}`)
    .digest('hex')
    .slice(0, 12);
}
```

```typescript
/**
 * The trust ledger is the agent's test record. It is the ONLY source of an
 * agent's band — the band is always re-derived from it, never carried by hand.
 *
 * Append-only. There is no update or delete path here, and there must never be.

export async function loadLedger(agentRole: string, version: string): Promise<LedgerEvent[]> {
  try {
    const rows = await db
      .select()
      .from(trustLedger)
      .where(and(eq(trustLedger.agentRole, agentRole), eq(trustLedger.agentVersion, version)))
      .orderBy(asc(trustLedger.timestamp), asc(trustLedger.id));

      bandBefore: r.fromBand as AutonomyBand,
      bandAfter: r.toBand as AutonomyBand,
      isSpendAction: r.isSpendAction,
      createdAt: r.timestamp,
```

```typescript
import type { PolicyRule } from '@/src/types';

export type BriefingMode = 'none' | 'full';

export interface BlockedProposal {
  actionType: string;
  payload: Record<string, unknown>;
  reason: string;
}

/**
 * Renders the policy for the agent's prompt.
 *
 * This is an EFFICIENCY measure, not a safety measure. A briefed agent wastes
 * fewer turns proposing things that will obviously be blocked — which matters,
 * because failed retries are the thing that actually grows an agent's context.
 *
 * It guarantees nothing. A briefed agent can still be argued round by a big
 * enough saving, hallucinate, or be prompt-injected. The evaluator is what makes
 * the rule a rule. Telling the model is a suggestion; the evaluator is the law.
 *
 * `none` is the demo setting: an uninformed agent genuinely proposes the cheaper
 * unapproved supplier, and the block is an honest test rather than a staged one.
 */
export function renderBriefing(rules: PolicyRule[], mode: BriefingMode): string {
  if (mode === 'none') return '';

  const vendors = rules
    .filter((r) => r.ruleType === 'VENDOR_APPROVAL' && r.appliesTo)
    .map((r) => r.appliesTo)
    .join(', ');

  const thresholds = rules
    .filter((r) => r.ruleType === 'SPEND_THRESHOLD' && r.thresholdValue !== undefined)
    .sort((a, b) => (a.thresholdValue ?? 0) - (b.thresholdValue ?? 0))
    .map((r) => `over ${r.thresholdValue} ${r.currency ?? 'GBP'} requires human approval`)
    .join('; ');

  const lines = ['=== POLICY (guidance) ==='];
  if (vendors) lines.push(`Approved vendors: ${vendors}. No other vendor may be used.`);
  if (thresholds) lines.push(`Spend limits: ${thresholds}.`);
  lines.push(
    'These are enforced by a separate deterministic evaluator regardless of what you propose. ' +
      'Proposing a violation will not succeed — it will be blocked and cost you standing.'
  );

  return lines.join('\n');
}

/**
 * What this mission already tried and had rejected.
 *
 * Without this, a blocked agent re-proposes the same rejected action forever:
 * it still wants the cheaper vendor, and nothing told it why it failed.
 */
export function renderBlockedProposals(blocked: BlockedProposal[]): string {
  if (blocked.length === 0) return '';

  const lines = ['=== ALREADY REJECTED THIS MISSION — do not repeat these ==='];

  for (const b of blocked) {
    lines.push(`- ${b.actionType} ${JSON.stringify(b.payload)}\n  REJECTED: ${b.reason}`);
  }

  return lines.join('\n');
}
```

```typescript
import * as dotenv from 'dotenv';
dotenv.config();

import * as fs from 'fs';
import { getAllDecisions } from '@/src/orchestrator/persistence';

/**
 * Turns the audit trail into a preference dataset.
 *
 * The evaluator is a deterministic reward function: it labels every proposal
 * ALLOW or BLOCK, for free, instantly, with no human in the loop. That labelling
 * is the expensive half of RLHF, and Mandate emits it as exhaust.
 *
 * Emits DPO-style pairs: within a mission, a BLOCKed proposal is the `rejected`
 * completion and the next non-blocked proposal for the same action type is the
 * `chosen` one.
 *
 * Fine-tuning itself (InstructLab / DPO on Granite) is future work. This proves
 * the corpus exists.
 */
interface PreferencePair {
  prompt: string;
  chosen: Record<string, unknown>;
  rejected: Record<string, unknown>;
  rejected_because: string;
  cited_rule: string | null;
}

async function main() {
  const decisions = await getAllDecisions();

  if (decisions === null) {
    console.error('Postgres is not running. Start it: npm run db:up');
    process.exit(1);
  }

  const byMission = new Map<string, typeof decisions>();
  for (const d of decisions) {
    if (!byMission.has(d.missionId)) byMission.set(d.missionId, []);
    byMission.get(d.missionId)!.push(d);
  }

  const pairs: PreferencePair[] = [];

  for (const [, steps] of byMission) {
    const ordered = [...steps].sort((a, b) => a.stepNumber - b.stepNumber);

    for (const blocked of ordered.filter((s) => s.verdict === 'BLOCK')) {
      // The chosen completion: a later, permitted proposal of the same action type.
      const chosen = ordered.find(
        (s) =>
          s.stepNumber > blocked.stepNumber &&
          s.actionType === blocked.actionType &&
          s.verdict !== 'BLOCK'
      );

      // Fall back to any permitted commercial action in the same mission.
      const fallback = ordered.find((s) => s.verdict !== 'BLOCK' && s.riskClass !== 'low');
      const good = chosen ?? fallback;
      if (!good) continue;

      pairs.push({
        prompt: `Mission: ${blocked.missionGoal}\nStep ${blocked.stepNumber}: propose a ${blocked.actionType} action.`,
        chosen: good.actionPayload as Record<string, unknown>,
        rejected: blocked.actionPayload as Record<string, unknown>,
        rejected_because: blocked.explanation ?? '',
        cited_rule: blocked.sourcePassage,
      });
    }
  }

  const out = 'data/training/preferences.jsonl';
  fs.mkdirSync('data/training', { recursive: true });
  fs.writeFileSync(out, pairs.map((p) => JSON.stringify(p)).join('\n') + '\n');

  console.log(`\n${decisions.length} decisions across ${byMission.size} missions`);
  console.log(`${pairs.length} preference pairs -> ${out}`);
  console.log('\nThe evaluator labelled every one of these. No human annotation.');
  console.log('Next: DPO / InstructLab fine-tune of Granite on this corpus.\n');
}

main().catch((e) => {
  console.error(e);
  process.exit(1);
});
```

```python
"""
Offline policy-document parsing with IBM Docling.

Runs ONCE, by hand, never at request time. Reads the policy PDFs in
data/policies/, converts them to structured text with Docling, and writes the
extracted rules to data/seed/ where `npm run db:seed` picks them up.

The output is human-reviewed before it is committed. A parser is not an
authority: no rule enters the policy library without a person confirming that
the source passage says what the rule claims it says.
"""
import json
import sys
from pathlib import Path

from docling.document_converter import DocumentConverter

POLICIES = Path("data/policies")
OUT = Path("data/seed")


def main() -> int:
    if not POLICIES.exists():
        print(f"No {POLICIES}/ — add the source policy PDFs first.", file=sys.stderr)
        return 1

    converter = DocumentConverter()
    pdfs = sorted(POLICIES.glob("*.pdf"))

    if not pdfs:
        print(f"No PDFs in {POLICIES}/", file=sys.stderr)
        return 1

    for pdf in pdfs:
        print(f"Parsing {pdf.name} ...")
        result = converter.convert(str(pdf))

        markdown = result.document.export_to_markdown()

        # The parsed text is written next to the seed file so a reviewer can
        # diff a rule's source_passage against what Docling actually read.
        parsed_path = OUT / f"{pdf.stem}.parsed.md"
        parsed_path.write_text(markdown, encoding="utf-8")

        print(f"  -> {parsed_path}")
        print(f"     {len(markdown)} chars, {len(result.document.texts)} text blocks")

    print(
        "\nDocling has parsed the source documents.\n"
        "Next: review each .parsed.md and confirm every sourcePassage in\n"
        "data/seed/*.json appears verbatim in the parsed text. A citation that\n"
        "does not match its source is worse than no citation.\n"
    )
    return 0


if __name__ == "__main__":
    sys.exit(main())
```
